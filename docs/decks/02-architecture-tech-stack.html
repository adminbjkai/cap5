03 / 05 Structure

Monorepo Layout

apps/web
React/Vite frontend — library, recording, watch page, transcript editing, AI enrichment review
apps/web-api
Fastify REST API — videos, uploads, library, auth, provider status, inbound/outbound webhooks
apps/worker
Background job processor — claims from PostgreSQL queue, runs process/transcribe/AI/cleanup/webhook jobs
apps/media-server
FFmpeg/ffprobe wrapper service — downloads raw, transcodes, generates thumbnail, uploads artifacts
packages/config
Zod-validated environment schema — canonical env contract, fails fast with clear error messages
packages/db
PostgreSQL pool + migrations — singleton per DATABASE_URL, max 20 connections, 30s idle timeout
packages/logger
Pino structured logging — redacts secrets, context propagation via child loggers, pretty-print mode
db/migrations
Schema source of truth — all table definitions and indexes versioned as sequential migration files
04 / 05 ADRs

Architecture Decisions

ADR-001: PostgreSQL is the queue — FOR UPDATE SKIP LOCKED plus leases, heartbeats, reclaim, and dead-lettering. No external broker needed.

ADR-002: Synchronous media orchestration — Worker calls media-server directly via POST /process. Simple pipeline, easy to reason about.

ADR-003: Shared HMAC shape — Inbound and outbound webhooks use the same timestamped HMAC format with delivery IDs.

ADR-004: Single-user JWT auth — Email/password with bcrypt, stateless HS256 JWT, httpOnly cookie transport. No multi-tenant overhead.

ADR-005: Runtime naming is cap5 — S3 bucket defaults to cap5, object routing uses /cap5/..., webhook media type uses application/cap5-webhook+json.
