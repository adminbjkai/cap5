01 / 05 Pipeline

End-to-End Processing Pipeline

1
Upload
Browser records screen or user uploads file. Single-part (≤100MB) or multipart (>100MB, 10MB chunks) signed S3 upload. Queues process_video.
➔
2
Process
Worker claims job. Media-server downloads raw from S3. FFmpeg transcodes to H.264/AAC MP4. Generates thumbnail. ffprobe extracts metadata. Uploads artifacts.
➔
3
Transcribe
Worker sends audio to Deepgram Nova-2. Returns segments with speaker diarization, timestamps, confidence. Stores transcript + WebVTT. Queues generate_ai.
➔
4
Enrich
Worker sends transcript to Groq Llama 3.3 70B. Generates title, summary, chapters, entities, action items, quotes. Multi-chunk synthesis for long transcripts.
➔
5
Deliver
Frontend polls GET /api/videos/:id/status. Renders video player, transcript, AI enrichments. Outbound webhooks fire at each milestone.
02 / 05 Media Processing

FFmpeg & Media Server

Processing Steps

Validate — confirm videoId is UUID format

Download — stream raw upload from S3 to /tmp/cap5-media/<videoId>/

Transcode — FFmpeg: H.264 veryfast, yuv420p, faststart, AAC 128k

Thumbnail — FFmpeg thumbnail filter, single best frame

Probe — ffprobe extracts duration, dimensions, FPS, audio presence

Upload — result MP4 + thumbnail to S3, cleanup temp files

FFmpeg Commands

# Normalize to MP4 ffmpeg -y -i <input> \ -map 0:v:0 -map 0:a:0? \ -c:v libx264 -preset veryfast \ -pix_fmt yuv420p -movflags +faststart \ -c:a aac -b:a 128k \ <output.mp4> # Generate thumbnail ffmpeg -y -i <result.mp4> \ -vf thumbnail -frames:v 1 \ <thumb.jpg> # Probe metadata ffprobe -v error -print_format json \ -show_streams -show_format <file>

S3 Key Patterns

result: videos/<videoId>/result/result.mp4 thumbnail: videos/<videoId>/thumb/screen-capture.jpg vtt: videos/<videoId>/transcript/transcript.vtt raw: uploads/<videoId>/<original-filename>
03 / 05 AI Providers

Deepgram & Groq Integration

Deepgram — Transcription

Model: Nova-2 (configurable via DEEPGRAM_MODEL)

Features: smart_format, punctuate, utterances, diarize, detect_language

Output: segments with { startSeconds, endSeconds, text, confidence, speaker }

Storage: segments_json + speaker_labels_json in transcripts table, VTT file in S3

Fallback: extracts audio via FFmpeg first; falls back to full video if needed

Groq — AI Enrichment

Model: Llama 3.3 70B Versatile (configurable via GROQ_MODEL)

Temperature: 0.3 — factual, low-creativity extraction

Response format: JSON object (structured output)

Multi-chunk: transcripts >24K chars split by paragraph boundaries

Synthesis: chunks processed individually, then combined via additional call

AI Output Fields

FieldDescriptionConstraints
titleAuto-generated descriptive title from transcript contentSingle string
summaryConcise overview of the video contentSingle string
key_pointsMain takeaways from the discussionArray of strings
chaptersTimestamped topic segments with titlesMax 12, deduped within 30s intervals
entitiesNamed people, orgs, tools, concepts mentionedSet-deduplicated across chunks
action_itemsTasks, assignments, or follow-ups identifiedArray with assignee/deadline fields
quotesNotable verbatim quotes from speakersMax 5 per video
04 / 05 Upload

Upload Strategies

Single-Part (≤ 100 MB)

1

POST /api/videos — create video + uploads row

2

POST /api/uploads/signed — get presigned PUT URL

3

PUT to signed URL — XHR upload with progress

4

POST /api/uploads/complete — queues process_video

Multipart (> 100 MB)

1

POST /api/uploads/multipart/initiate — start session

2

POST /api/uploads/multipart/presign-part — per 10MB chunk

3

PUT each chunk — sequential upload with ETag tracking

4

POST /api/uploads/multipart/complete — ETags array, queues job
